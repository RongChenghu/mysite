# 从Excel到智能枢纽：我如何将供应链从“表格丛林”搬进系统轨道

当我第一次接手远茂供应链项目时，眼前的“系统”是一堆命名诡异的Excel：
📁 《库存表最终版v3》
📁 《最新价格_李总改过》
📁 《订单2021汇总_new》

在很多人眼中这是混乱，而我看到的却是**未经雕琢的业务逻辑原型**。

------

### 一、破局：从“表格丛林”中提炼系统骨架

我没有立即写PRD，而是做了一次“业务考古”——从上百个文件中还原出四个核心实体：

**商品（SPU/SKU） → 价格 → 库存 → 订单**

这四条链路，构成了供应链的“生命循环”。比如：

- 价格策略依附于商品生命周期
- 库存变动与订单状态实时联动
- 订单流转触发全链路数据更新

------

### 二、重构：把人工逻辑转化为系统规则

我把Excel中的“人肉逻辑”逐步翻译成系统语言：

| Excel中的操作      | 转化为系统规则       |
| :----------------- | :------------------- |
| 手动修改单价单元格 | → 多级价格策略引擎   |
| 颜色标记低库存     | → 自动安全库存预警   |
| 复制粘贴订单信息   | → 前端一键生成订单流 |

最难的其实不是技术，而是**让业务团队相信系统比Excel可靠**。我带着他们对比数据：

- 系统上线后订单错误率**下降60%**
- 库存周转时长**缩短至原1/3**
- 首次实现**实时利润看板**

当采购经理看到系统自动提醒“A商品库存低于安全线，建议立即补货”时，Excel时代正式终结。

------

### 三、启示：数字化转型的本质是思维升级

这次经历让我深刻认识到：
**从Excel到系统的距离，不是工具迭代，而是从“人驱动”到“数据驱动”的认知跃迁**

真正的数字化转型，起点永远不是技术方案，而是**用结构化思维重构业务逻辑**。当你把Excel中的IF函数变成系统规则，把手动粘贴变成API接口，把经验判断变成数据模型——业务才真正拥有了智能化的根基。

（后来这套“实体提炼→规则转化→数据驱动”的方法，被我用在多个传统行业数字化项目中，成为打破“表格依赖”的利器。）
